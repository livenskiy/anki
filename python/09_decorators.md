# Basic
Механика синтаксиса: Что на самом деле происходит, когда мы пишем @decorator над функцией? В какой момент вызывается сам код декоратора — в момент определения функции или в момент её вызова?
Синтаксический сахар: Как переписать применение декоратора @trace к функции def func(), не используя символ @?
Классовые декораторы: Можно ли использовать класс в качестве декоратора? Что для этого должно быть реализовано в классе (какой магический метод)?

# Advanced
Декораторы с параметрами: Если мы пишем @trace('LOG'), сколько уровней вложенных функций должно быть в коде декоратора? Что именно возвращает вызов trace('LOG') до того, как он применится к функции?
Сохранение метаданных: Когда мы оборачиваем функцию в декоратор, её имя (__name__) и документация (__doc__) могут «стереться», заменившись данными обертки. Как это правильно исправить?
Декорирование классов: Что происходит, если применить декоратор не к функции, а к классу? Что он получает на вход и что должен вернуть?

# Практическое задание (Coding Challenge)
Задача: Напиши декоратор с параметром @repeat(n), который вызывает декорируемую функцию n раз.
Декоратор должен работать с любыми аргументами функции (*args, **kwargs).
Должен возвращать результат последнего выполнения.
Используй functools.wraps, чтобы не потерять имя функции.
Пример использования:
python
@repeat(3)
def say_hello(name):
    print(f"Hello, {name}!")
    return "Done"

# Вызов say_hello("Alice") должен напечатать приветствие 3 раза
# и вернуть "Done"
